# Revenue Prototype

Простой фронтенд-прототип скоринга / модели дохода для кейса «Альфа». Минимальный стек: чистый HTML + CSS + JavaScript.

Структура:
- `index.html` — форма для ввода данных клиента (POST /finance)
- `model.html` — страница результатов: опрос по `task_id` (GET /task/{task_id}) и отображение dict с результатами
- `css/styles.css` — стили в палитре красный/чёрный/белый
- `js/main.js` — логика отправки, опроса, отрисовки

Как запустить:
- Откройте `index.html` в браузере (подходит для базовой проверки).
- Рекомендуется поднять статический сервер из корня проекта:

```bash
cd /Users/a1/Code/Frontend/revenue-prototype
python3 -m http.server 8000
# затем откройте http://localhost:8000
```

Интеграция с бекендом:
- POST `/finance` — фронт отправляет JSON с полями формы. Ожидается ответ JSON с ключом `task_id`.
- GET `/task/{task_id}` — фронт опрашивает этот эндпоинт до получения статуса `done` и результата. Ожидается ответ вида `{status: 'done', result: {...}}` либо прямо dict результата.

Локальная разработка / mock mode:
- Если POST или GET не доступны (локальная проверка), фронт перейдёт в mock-режим: использует `task_id` `d6c6402e-7423-46ed-ada0-44bb923613ee` и сохраняет mock-результат в `localStorage`.
